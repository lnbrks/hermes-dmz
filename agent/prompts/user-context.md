# User Context

Free-form notes for the agent: anything I want it to know. Sender rules,
subscriptions, relationships, work context, ad-hoc preferences. The agent
reads this every run. I edit it between runs to teach the agent (often in
response to its `## Questions` section in the previous notification).

Treat these as strong defaults to apply with judgment, not absolute hard
rules. If something here conflicts with obvious common sense for a given
email, use judgment and flag it under `## Worth a look` or `## Questions`.

---

## Sender-Specific Rules

## Always trash (high-volume marketing)

- Wayfair, Urban Outfitters, Cider, Aritzia, BNTO, Crate & Barrel,
  Modern Citizen, Huel (promotional content; receipts → archive),
  Quince, Free People, Pepper.
- All political fundraising senders.

## Always archive (transactional residue)

- USPS shipping notifications.
- Venmo, PayPal notifications (received/sent confirmations).
- Schwab, Fidelity statements/notifications.
- Firebase App Distribution notifications.

## Senders with nuance

- **Hannah Lim** — real friend. Personal emails stay in inbox. Blog updates
  and calendar invites → archive.
- **Caltech Alumni Relations** — all solicitations, archive.
- **One Medical** — "Message from X" notifications with no content → trash if
  >7 days old. "Action Item from X" → trash if >60 days old. Appointment
  confirmations and prescription records → keep until the appointment is past,
  then archive.
- **MyChart** — "You have a new message" notifications → trash if >30 days old.
- **Medfinder** — "We found your medications" → archive (done).
- **BNTO subscription renewal reminders** → archive (recurring nag).
- **ClickPay payment reminders** → archive if >30 days old (rent handled).
- **Welcome emails** — keep the first one for an account worth remembering;
  archive after a year (knowing the account exists is enough).
